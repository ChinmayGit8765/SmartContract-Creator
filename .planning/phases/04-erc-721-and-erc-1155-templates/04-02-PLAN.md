## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user input → wizard validators | Royalty recipient (`0x…`), royalty bps (0-10000), contract name (Solidity identifier), symbol (1-11 ASCII), baseUri (no whitespace) all enforced via `validate:` callbacks on @clack text prompts |
| wizard opts → erc721.print | Pass-through 1:1 via the typed `Erc721Opts` → `ERC721Options` mapping in generate.ts; no string interpolation of user input into Solidity outside the royalty post-process |
| Royalty post-process | Anchor-located injection of OZ-provided primitives (`ERC2981`, `_setDefaultRoyalty`); receiver is a hex literal validated by regex; feeNumerator is a parsed number 0..10000; no free-form user string flows into Solidity |
| smartc → @openzeppelin/wizard | Trusted, pinned; legitimacy audited in Phase 2; no new install |
